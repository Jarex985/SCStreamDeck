# Contributing to Star Citizen Stream Deck

Thank you for your interest in contributing to the Star Citizen Stream Deck plugin! I appreciate all contributions, whether it's bug reports, feature requests, or code changes.

## How to Contribute

### Reporting Issues
- Use the [GitHub Issues](https://github.com/Jarex985/SCStreamDeck/issues) page to report bugs or suggest features.
- Provide as much detail as possible, including steps to reproduce, expected behavior, and your environment (OS, Stream Deck software version, etc.).

### Submitting Changes
1. Fork the repository.
2. Create a new branch for your changes: `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/your-bugfix-name`.
3. Make your changes, ensuring they follow the coding guidelines (see below).
4. Write or update unit tests if applicable.
5. Commit your changes: `git commit -m "Description of changes"`.
6. Push to your fork: `git push origin your-branch-name`.
7. Submit a Pull Request (PR) to the main repository.

**Pull Request Guidelines**: Please keep pull requests focused on a single feature or bug fix to facilitate easier reviews. If your changes are extensive, consider splitting them into multiple smaller PRs. Large or unfocused PRs may be requested to be revised or declined.

### Coding Guidelines
- Follow C# best practices and the guidelines outlined in the project.
- Use meaningful variable and method names.
- Ensure code is well-commented.
- Run unit tests before submitting.

### Testing
- Run the existing unit tests to ensure no regressions.
- Add new tests for new features or bug fixes.
- Ensure tests pass before submitting your PR.
